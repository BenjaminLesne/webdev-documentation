## Tickets

### Story

```markdown
### Résumé

1. **En tant que** [utilisateur concerné par la story]
2. **Je veux** [but de la story]
3. **afin que** [raison de la story] (facultatif)

ex: Cela peut alors devenir "En tant qu'utilisateur gérant mes propriétés, je veux des notifications lors de l'ajout ou de la suppression d'images".


### Critères d'acceptation
(précisez le rôle)

1. [Si je fais A.]
1. [B devrait arriver.]


### Ressources:

* Maquettes : [Voici l’URL de la maquette 1, l’URL de la maquette 2, etc]
```

## Others
### Merge request
```markdown
# Toutes les cases doivent être cochées avant la fusion.

- [ ] Le commit suit la convention : [type]:[ticket number] [description], 1ère ligne <= 50 cacaractères.
- [ ] Le code est expliqué dans la description de la merge request
- [ ] Les travaux temporaires sont supprimés (code commenté, console logs, etc).
- [ ] Des tests ont été ajoutés ou mis à jours lorsque nécessaire.
- [ ] Une vidéo montrant que le code fonctionne avec la console navigateur affichée est ajoutée à la section “Vidéos ou screenshots”([logiciel pour enregistrer](https://obsproject.com/))

# De quoi s’agit-il ?

%{all_commits}

# Videos ou screenshots
```
