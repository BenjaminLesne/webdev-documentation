# Configuration

If some tsconfig.json options are set to true, others are automatically set to true as well. This means that the user doesn't need to set them.

verbatimModuleSyntax: true -> isolatedModules: true

strict: true -> noImplicitAny: true, noImplicitThis: true, strictNullChecks: true, strictFunctionTypes: true, strictBindCallApply: true, strictPropertyInitialization: true, noImplicitReturns: true, useUnknownInCatchVariables: true, strictBuiltinIteratorReturn: true

esModuleInterop: true -> allowSyntheticDefaultImports: true

module: "node16" -> moduleResolution: "node16"

module: "node18" -> moduleResolution: "node18"

module: "nodenext" -> moduleResolution: "nodenext"

target: "ES2022" -> lib: ["ES2022", "DOM", "DOM.Iterable"] (if lib is not explicitly set)

[source](https://github.com/mattpocock/ai-hero-cli-archived/blob/main/prompts/hows-my-tsconfig.md)

Use the archived [AI hero repo](https://github.com/mattpocock/ai-hero-cli-archived/blob/main/prompts/hows-my-tsconfig.md) to ask AI about the tsconfig.json file

### Required

- [ts-reset](https://github.com/mattpocock/ts-reset)

### Optional

- [string-ts](https://github.com/gustavoguichard/string-ts)

### Example

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    },
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowJs": true,
    "moduleDetection": "force",
    "verbatimModuleSyntax": true,

    /* Bundler mode */
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",

    /* Linting */
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitOverride": true
  },
  "include": ["src", "tests"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```
